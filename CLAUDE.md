# CLAUDE.md - IHC Image Viewer

## Project Overview

R Shiny application for viewing IHC (immunohistochemistry) whole-slide images using [OpenSeadragon](https://openseadragon.github.io/) v6.0.2. Designed for deployment on Posit Connect.

## Current Status

**Working (Step 3 only):** Local image viewing with OpenSeadragon.

**Not yet implemented:**

- Step 2: S3 image download to local disk


## Architecture

### How it works

1. **app.R** defines the Shiny server. On startup it registers `addResourcePath("images", ...)` and `addResourcePath("dzi", ...)` so the browser can fetch image files.
2. **template.html** is the full HTML page (Catppuccin dark theme). It loads OpenSeadragon from CDN and defines three Shiny custom message handlers:
   - `loadImage` — creates an OpenSeadragon viewer with either `{type:"image", url:...}` (simple) or a DZI URL
   - `clearViewer` — destroys the viewer and shows empty state
   - `showToast` / `updateFileCount` — UI notifications
3. User clicks an image in the sidebar → JS sends `Shiny.setInputValue("select_image", filename)` → server validates the path (directory traversal protection via `safe_path()`), determines format, sends the appropriate `loadImage` message back.

### Image format handling

| Format | How it's served | Notes |
|--------|----------------|-------|
| PNG, JPG, GIF, BMP, WEBP | OpenSeadragon Simple Image mode | Loaded whole; warns if >100 MB |
| SVS, TIFF, NDPI, MRXS, SCN, BIF | Must convert to DZI first | Use `convert_to_dzi.sh` (requires `vips`) |
| .dzi + _files/ | OpenSeadragon DZI tile source | Tiled pyramid, efficient for any size |

### Key design decisions

- **Custom HTML template** (`htmlTemplate`) instead of `fluidPage` — full control over layout, dark theme, no Bootstrap overhead.
- **Event delegation** for image list clicks — sidebar items use `data-file` attributes, single `document.addEventListener("click")` handler. No inline `onclick`.
- **Path validation** — `safe_path()` calls `basename()` then `normalizePath()` to block directory traversal attacks.
- **OpenSeadragon CDN** — loaded from `cdn.jsdelivr.net/npm/openseadragon@6.0.2`. For offline/air-gapped deployment, download to `www/` and update the `<script src>` in template.html.

## Setup on a New Machine

```bash
cd IHC_viewer
bash setup.sh
```

`setup.sh` handles: R check → libuv detection (uses bundled fallback) → vips check (optional) → `renv::restore()` from lockfile → verification.

### Manual setup

```bash
# 1. Install renv + restore packages
R -e 'install.packages("renv"); Sys.setenv(USE_BUNDLED_LIBUV="1"); renv::restore()'

# 2. (Optional) Install vips for WSI conversion
conda install -c conda-forge libvips

# 3. Run
Rscript -e 'shiny::runApp(".", port=3838, host="0.0.0.0")'
```

### System dependencies

- **R >= 4.0** (tested on 4.1.2)
- **libuv** — needed to compile the `fs` R package. If not available as system library, set `USE_BUNDLED_LIBUV=1` before `renv::restore()` (setup.sh does this automatically).
- **libvips** (optional) — only needed for converting WSI files (SVS/TIFF/NDPI) to DZI tiles. Install via `conda install -c conda-forge libvips` or `apt install libvips-tools`.

## R Dependencies

All managed via `renv`. The only direct dependency is `shiny` (v1.13.0). Full transitive dependency tree (30 packages) is locked in `renv.lock`.

To add a new package:
```r
renv::install("packagename")
renv::snapshot()   # updates renv.lock
```

## Converting WSI Images

```bash
# Single file
./convert_to_dzi.sh images/slide001.svs

# All images in a directory
./convert_to_dzi.sh images/

# Manual vips command
vips dzsave images/slide001.svs dzi_tiles/slide001 --tile-size 254 --overlap 1
```

Output: `dzi_tiles/slide001.dzi` + `dzi_tiles/slide001_files/` (tile pyramid). The app auto-detects DZI availability per image.

### Step 2: S3 Download to Local Disk

- On image selection, download from S3 to `images/` directory if not already cached locally.
- Example:
  ```r
  s3$download_file(Bucket = "bucket", Key = "path/to/image.svs",
                   Filename = file.path("images", "image.svs"))
  ```
- Show a progress indicator during download (large WSI files can be >1 GB).
- Implement local cache: check if file already exists + matches size/ETag before re-downloading.

### Step 2.5 (Optional): Preprocessing

- If SVS/TIFF files need DZI conversion before viewing, trigger `convert_to_dzi.sh` or call `vips dzsave` via `system2()` after download.
- Could also run preprocessing (e.g., tissue detection, thumbnail generation) at this stage.

### Step 3: Posit Connect Deployment

- `rsconnect::deployApp(".", appTitle = "IHC Image Viewer")`
- Ensure the Posit Connect server has:
  - Network access to the S3 bucket
  - AWS credentials configured (env vars or IAM role)
  - `libvips` installed (if DZI conversion is needed server-side)
  - Sufficient disk space for cached images
- The `renv.lock` ensures reproducible package installation on Posit Connect.
- Consider `persistent` storage for the images/ and dzi_tiles/ directories (not ephemeral app storage).

### Security Notes

- S3 credentials should use IAM roles on Posit Connect, not hardcoded keys.
- `safe_path()` in app.R prevents directory traversal — keep this when adding S3 download paths.
- Sanitize S3 object keys before using as local filenames.
